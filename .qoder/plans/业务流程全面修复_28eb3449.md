# 业务流程全面修复计划

## 审查发现的问题清单

### 核心交易链路 (订单/支付/库存)
1. **createOrder 使用硬编码价格** - `order.service.ts:94-95` 用 `qty * 100` 占位，未查询真实商品价格
2. **下单无库存扣减/销量增加** - 流程图要求"扣减 PRODUCT.stock / 增加 sales"，代码完全缺失
3. **无支付流程** - 缺少支付API和支付成功回调，流程图要求 INSERT PAYMENT 并更新订单状态
4. **取消订单不回滚库存** - `cancelOrder` 只改状态，不恢复 stock/sales
5. **确认收货无后续触发** - `confirmReceive` 缺少：积分增加、分佣计算、消费流水、会员升级检查

### 分佣结算
6. **无自动分佣计算** - 支付成功后不追溯邀请链计算佣金，不写入 COMMISSION_LOG
7. **提现无余额校验和冻结** - `applyWithdraw` 不检查余额 >= 10，不冻结 available_balance

### 课程购买
8. **课程购买不触发分佣资格** - 购买2999/5999课程后应设置 USER.level 和创建 USER_COMMISSION，代码缺失

### 商家入驻
9. **审核通过不自动创建商家** - `reviewApply` 仅更新状态，不将申请信息同步到 CITY_MERCHANT

### 退款售后
10. **缺少退款流程实现** - 无 REFUND 相关的创建/审核/退款处理 API

### 积分与会员
11. **签到积分不同步 USER.points** - `signIn` 写 POINTS_LOG 但不更新 USER.points 字段
12. **缺少会员自动升级逻辑** - 无地方检查 total_spent 是否达到 MEMBER_LEVEL.threshold

---

## 修复任务

### Task 1: 修复订单创建 - 真实价格 + 库存扣减
**文件**: `server/src/module/pet/order/order.service.ts`
- `createOrder` 方法中 JOIN product 表获取真实价格
- 支持 flash_price 优先级（限时特供价 > 正常价）
- 扣减 product.stock，增加 product.sales
- 库存不足时拒绝下单
- 正确计算 totalAmount / originalAmount / discountAmount

### Task 2: 添加支付流程
**文件**: `server/src/module/pet/order/order.service.ts` + `order.controller.ts`
- 新增 `payOrder` 方法：创建 PAYMENT 记录，更新 ORDER.status = 'paid'
- 支付成功后触发：积分计算、分佣计算、消费流水记录
- 新增 `payNotify` 模拟支付回调（因无真实微信对接）

### Task 3: 修复取消订单 - 库存回滚
**文件**: `server/src/module/pet/order/order.service.ts`
- `cancelOrder` 中查询 ORDER_ITEM，回滚 product.stock 和 sales

### Task 4: 修复确认收货 - 触发完整后续链路
**文件**: `server/src/module/pet/order/order.service.ts`
- `confirmReceive` 中增加：
  - 更新 USER.total_spent
  - 调用积分服务（按 member_level.points_rate 计算）
  - 调用分佣服务（追溯邀请链计算佣金）
  - 写入 CONSUMPTION_LOG
  - 检查会员等级升级

### Task 5: 修复分佣自动计算
**文件**: `server/src/module/pet/commission/commission.service.ts`
- 新增 `calculateCommission(orderId, userId, totalAmount)` 方法
- 通过 USER_INVITE 向上追溯邀请链
- 按 COMMISSION_TIER 差额费率计算每级佣金
- 写入 COMMISSION_LOG，更新 USER_COMMISSION.pending_amount

### Task 6: 修复提现余额校验与冻结
**文件**: `server/src/module/pet/commission/commission.service.ts`
- `applyWithdraw` 中校验 available_balance >= 10
- 校验 amount <= available_balance
- 扣减 available_balance，增加 frozen_amount

### Task 7: 修复提现审核 - 余额回滚
**文件**: `server/src/module/pet/commission/commission.service.ts`
- `auditWithdraw` 拒绝时回滚 available_balance，扣减 frozen_amount
- 通过时扣减 frozen_amount

### Task 8: 修复课程购买触发分佣资格
**文件**: `server/src/module/pet/course/course.service.ts`
- `purchaseCourse` 中根据课程 tier (pro/partner) 设置 USER.level
- 创建/更新 USER_COMMISSION 记录，关联对应 tier_id

### Task 9: 修复商家入驻审核自动创建商家
**文件**: `server/src/module/pet/merchant/merchant.service.ts`
- `reviewApply` 审核通过时自动将 MERCHANT_APPLY 信息同步创建 CITY_MERCHANT

### Task 10: 添加退款售后流程
**文件**: `server/src/module/pet/order/order.service.ts` + 新增 refund entity 检查
- 新增 createRefund / auditRefund 方法
- 退款审核通过后回滚积分、更新订单状态

### Task 11: 修复签到积分同步 USER.points
**文件**: `server/src/module/pet/finance/finance.service.ts`
- `signIn` 中同步更新 USER.points += 10

### Task 12: 添加会员自动升级检查
**文件**: `server/src/module/pet/order/order.service.ts` (在 confirmReceive 中调用)
- 新增 `checkMemberUpgrade(userId)` 方法
- 查询 MEMBER_LEVEL，比对 total_spent 与 threshold
- 只升不降

### Task 13: 修复订单模块依赖注入
**文件**: `server/src/module/pet/order/order.module.ts`
- 注入 ProductEntity、UserEntity、PointsLogEntity、CommissionService 等依赖

### Task 14: 编写 E2E 测试脚本验证完整流程
**文件**: `server/test/business-flow.e2e-spec.ts`
- 测试完整链路：创建商品 -> 加购物车 -> 下单 -> 支付 -> 发货 -> 收货 -> 验证积分/分佣/会员变化
- 测试分佣链路：创建邀请关系 -> 被邀请人消费 -> 验证佣金计算
- 测试提现流程
- 测试课程购买触发分佣资格
- 测试库存不足拒绝下单
